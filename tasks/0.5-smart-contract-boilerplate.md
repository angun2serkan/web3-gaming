## Amac

Solana blockchain uzerinde calisacak escrow smart contract'inin gelistirme ortamini ve iskelet yapisini kurmak. Anchor Framework ile proje yapisi olusturulacak, Solana CLI ve local validator yapilandirilacak, escrow programinin temel account yapilari ve instruction imzalari tanimlanacak.

Bu task tamamlandiginda:
- `anchor build` ile program derlenebilir durumda olacak
- `anchor test` ile local validator uzerinde temel testler calisacak
- Escrow programinin account yapilari (`GameRoom`, `PlayerEntry`, `EscrowVault`) tanimlanmis olacak
- SPL Token (USDT) entegrasyonu icin gerekli bagimliliklar kurulu olacak
- IDL (Interface Definition Language) dosyasi otomatik uretilecek ve frontend/backend tarafindan tuketilebilir olacak
- Turborepo uzerinden `pnpm run build` ile contracts paketi de derlenebilecek

> **Not:** Bu task'ta is mantigi (gercek escrow kilitleme, payout hesabi vb.) yazilmaz. Sadece iskelet yapi ve toolchain kurulumu yapilir. Is mantigi Faz 2 Sprint 5'te gelistirilecektir.

## Mimari Karar: Neden Anchor Framework?

| | Vanilla Solana (Raw Rust) | Anchor Framework |
|---|---------------------------|-----------------|
| **Gelistirme hizi** | Dusuk — her account deserializasyonu ve constraint kontrolu elle yazilir | Yuksek — makrolar ile otomatik |
| **Guvenlik** | Elle constraint kontrolu, hata riski yuksek | Otomatik account dogrulama, standart guvenlik kontrolleri |
| **Test** | Karmasik — `solana-program-test` ile low-level | Basit — `anchor test` ile Mocha/TypeScript entegrasyonu |
| **IDL uretimi** | Yok — API dokumantasyonu elle yazilir | Otomatik — JSON IDL dosyasi uretilir, client SDK otomatik olusur |
| **Ekosistem** | Sinirli community kaynagi | Genis community, hazir ornekler, iyi dokumantasyon |
| **SPL Token** | Elle CPI (Cross-Program Invocation) yazilir | `anchor-spl` crate ile hazir token islemleri |

> **Sonuc:** Escrow programi icin Anchor Framework tercih edilir. Gelistirme suresi kisalir, guvenlik riskleri azalir ve IDL sayesinde frontend/backend entegrasyonu kolaylasir.

#### 5.2 `contracts/programs/escrow/src/state.rs`

```rust
use anchor_lang::prelude::*;

/// Bir oyun odasinin on-chain state'i.
/// Her oda icin bir PDA olusturulur.
#[account]
#[derive(InitSpace)]
pub struct GameRoom {
    /// Oda olusturucusu / platform authority
    pub authority: Pubkey,

    /// Odanin benzersiz kimlik bilgisi (backend room ID ile eslesir)
    #[max_len(64)]
    pub room_id: String,

    /// Giris ucreti (lamports cinsinden, SPL Token decimals dahil)
    pub entry_fee: u64,

    /// Minimum oyuncu sayisi (varsayilan: 5)
    pub min_players: u8,

    /// Maksimum oyuncu sayisi
    pub max_players: u8,

    /// Mevcut oyuncu sayisi
    pub current_players: u8,

    /// Toplam havuz miktari (locked USDT)
    pub total_pool: u64,

    /// Oda durumu: 0=Waiting, 1=InProgress, 2=Completed, 3=Cancelled
    pub status: u8,

    /// Provably fair: round oncesi commit edilen seed hash
    pub committed_seed_hash: [u8; 32],

    /// PDA bump seed
    pub bump: u8,

    /// Olusturulma zamani (Unix timestamp)
    pub created_at: i64,
}

/// Bir oyuncunun odaya katilim kaydi.
/// Her oyuncu-oda cifti icin bir PDA olusturulur.
#[account]
#[derive(InitSpace)]
pub struct PlayerEntry {
    /// Oyuncunun cuzdan adresi
    pub player: Pubkey,

    /// Iliskili oda
    pub game_room: Pubkey,

    /// Odenen giris ucreti
    pub amount_paid: u64,

    /// Oyuncu durumu: 0=Active, 1=Eliminated, 2=Winner
    pub status: u8,

    /// PDA bump seed
    pub bump: u8,
}

/// Escrow vault konfigurasyonu.
/// Platform genelinde tek bir vault veya oda bazinda ayri vault olabilir.
#[account]
#[derive(InitSpace)]
pub struct EscrowVault {
    /// Vault authority (program PDA)
    pub authority: Pubkey,

    /// Iliskili oda
    pub game_room: Pubkey,

    /// Vault'taki toplam kilitli miktar
    pub total_locked: u64,

    /// PDA bump seed
    pub bump: u8,
}
```

**Notlar:**
- `#[derive(InitSpace)]` — Anchor otomatik olarak account boyutunu hesaplar
- `#[max_len(64)]` — String alanlari icin maksimum byte uzunlugu
- `status` alanlari `u8` olarak tanimlanir — Anchor enum'lari da kullanilabilir ancak basitlik acisindan su an `u8` tercih edildi
- Her account'ta `bump` alani — PDA dogrulama icin gerekli
- `GameRoom.committed_seed_hash` — provably fair mekanizmasi icin seed hash on-chain kaydi

```typescript
import * as anchor from "@coral-xyz/anchor";
import { Program } from "@coral-xyz/anchor";
import { Escrow } from "../target/types/escrow";
import { expect } from "chai";

describe("escrow", () => {
  // Anchor test provider'i yapilandir
  const provider = anchor.AnchorProvider.env();
  anchor.setProvider(provider);

  const program = anchor.workspace.Escrow as Program<Escrow>;

  it("Program basariyla deploy edildi", async () => {
    // Program ID'nin gecerli oldugunu dogrula
    const programInfo = await provider.connection.getAccountInfo(
      program.programId
    );
    expect(programInfo).to.not.be.null;
  });

  it("Oda basariyla olusturulabilir", async () => {
    const roomId = "test-room-001";
    const entryFee = new anchor.BN(1_000_000); // 1 USDT (6 decimals)
    const minPlayers = 5;
    const maxPlayers = 50;

    // GameRoom PDA'sini hesapla
    const [gameRoomPda] = anchor.web3.PublicKey.findProgramAddressSync(
      [Buffer.from("game_room"), Buffer.from(roomId)],
      program.programId
    );

    // initialize_room instruction'ini cagir
    await program.methods
      .initializeRoom(roomId, entryFee, minPlayers, maxPlayers)
      .accounts({
        authority: provider.wallet.publicKey,
        gameRoom: gameRoomPda,
        systemProgram: anchor.web3.SystemProgram.programId,
      })
      .rpc();

    // Olusturulan account'i oku ve dogrula
    const gameRoom = await program.account.gameRoom.fetch(gameRoomPda);
    expect(gameRoom.roomId).to.equal(roomId);
    expect(gameRoom.authority.toBase58()).to.equal(
      provider.wallet.publicKey.toBase58()
    );
    expect(gameRoom.status).to.equal(0); // WAITING
  });
});
```

### 8. IDL Paylasim Stratejisi

Anchor, program derlendiginde `target/idl/escrow.json` ve `target/types/escrow.ts` dosyalarini otomatik uretir. Bu dosyalar frontend ve backend'in programla etkilesimi icin gereklidir.

#### 8.1 IDL Dosyasi Nedir?

IDL (Interface Definition Language), smart contract'in API tanimidir. Icerir:
- Tum instruction'larin isimleri, parametreleri ve account gereksinimleri
- Tum account yapilari ve alanlari
- Tum hata kodlari ve mesajlari
- Program ID

#### 8.2 Paylasim Yontemi

```
anchor build
    |
    v
target/idl/escrow.json          ← JSON IDL (API tanimi)
target/types/escrow.ts          ← TypeScript tip tanimlari
    |
    ├──→ frontend/src/lib/idl/escrow.json     (kopyala)
    └──→ backend/src/lib/idl/escrow.json      (kopyala)
```

#### 8.3 IDL Dizinlerini Olustur

