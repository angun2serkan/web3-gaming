## Mimari Karar: Neden Iki Veritabani?

| | PostgreSQL | Redis |
|---|-----------|-------|
| **Tip** | Disk-tabanli iliskisel DB | In-memory key-value store |
| **Latency** | ~1-5ms | ~0.1ms (sub-millisecond) |
| **Dayaniklilik** | Kalici (ACID) | Gecici (varsayilan) |
| **Kullanim** | Oyun gecmisi, odeme kayitlari, cuzdan adresleri, oda istatistikleri | Aktif oyun state'i, oda verileri, oyuncu pozisyonlari, round state, session'lar |
| **Gerekcesi** | Finansal islemlerin ve oyun gecmisinin kalici, tutarli ve sorgulanabilir olmasi gerekir | Real-time oyun verisi sub-millisecond erisim gerektirir, kalicilik gerekmez |

> **Kural:** Ayni isi iki veritabanina yaptirma. Redis = hizli ve gecici, PostgreSQL = kalici ve guvenilir.

## Teknoloji Secimleri

| Teknoloji | Versiyon | Rol | Gerekcesi |
|-----------|----------|-----|-----------|
| **Docker Compose** | v2 | Container orkestrasyonu | PostgreSQL ve Redis'i tek komutla ayaga kaldirir. Gelistirici ortami standardize eder. |

| **ioredis** | ^5.x | Redis client | Cluster destegi, pipeline/batch, Lua scripting. Socket.io Redis adapter ile uyumlu. |

#### 3.2 `backend/prisma/schema.prisma`

```prisma
generator client {
  provider = "prisma-client-js"
  output   = "../node_modules/.prisma/client"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// --- Oyuncu ---

model Player {
  id            String   @id @default(cuid())
  walletAddress String   @unique @map("wallet_address")
  displayName   String?  @map("display_name")
  createdAt     DateTime @default(now()) @map("created_at")
  updatedAt     DateTime @updatedAt @map("updated_at")

  // Iliskiler
  roomPlayers  RoomPlayer[]
  transactions Transaction[]

  @@map("players")
}

// --- Oda ---

model Room {
  id        String     @id @default(cuid())
  type      RoomType
  entryFee  Decimal    @map("entry_fee") @db.Decimal(18, 9)
  status    RoomStatus @default(WAITING)
  minPlayers Int       @default(5) @map("min_players")
  maxPlayers Int?      @map("max_players")
  createdAt DateTime   @default(now()) @map("created_at")
  closedAt  DateTime?  @map("closed_at")

  // Iliskiler
  players RoomPlayer[]
  games   Game[]

  @@index([status])
  @@index([type, status])
  @@map("rooms")
}

enum RoomType {
  QUICK
  GRAND
  PRIVATE
}

enum RoomStatus {
  WAITING
  IN_PROGRESS
  COMPLETED
  CANCELLED
}

// --- Oda-Oyuncu Iliskisi ---

model RoomPlayer {
  id       String           @id @default(cuid())
  roomId   String           @map("room_id")
  playerId String           @map("player_id")
  joinedAt DateTime         @default(now()) @map("joined_at")
  status   RoomPlayerStatus @default(ACTIVE)

  room   Room   @relation(fields: [roomId], references: [id])
  player Player @relation(fields: [playerId], references: [id])

  @@unique([roomId, playerId])
  @@index([roomId])
  @@index([playerId])
  @@map("room_players")
}

enum RoomPlayerStatus {
  ACTIVE
  ELIMINATED
  WINNER
  DISCONNECTED
}

// --- Oyun ---

model Game {
  id         String     @id @default(cuid())
  roomId     String     @map("room_id")
  format     GameFormat
  status     GameStatus @default(IN_PROGRESS)
  poolAmount Decimal    @map("pool_amount") @db.Decimal(18, 9)
  commission Decimal    @default(0) @db.Decimal(18, 9)
  startedAt  DateTime   @default(now()) @map("started_at")
  endedAt    DateTime?  @map("ended_at")

  room   Room    @relation(fields: [roomId], references: [id])
  rounds Round[]

  @@index([roomId])
  @@index([status])
  @@map("games")
}

enum GameFormat {
  FORMAT_A  // Spinning Arrow
  FORMAT_B  // Snake Arena
}

enum GameStatus {
  IN_PROGRESS
  COMPLETED
  CANCELLED
}

// --- Round (Format A) ---

model Round {
  id              String      @id @default(cuid())
  gameId          String      @map("game_id")
  roundNumber     Int         @map("round_number")
  protectedNumber Int?        @map("protected_number")
  arrowResult     Int?        @map("arrow_result")
  status          RoundStatus @default(IN_PROGRESS)
  startedAt       DateTime    @default(now()) @map("started_at")
  endedAt         DateTime?   @map("ended_at")

  game Game @relation(fields: [gameId], references: [id])

  @@unique([gameId, roundNumber])
  @@index([gameId])
  @@map("rounds")
}

enum RoundStatus {
  IN_PROGRESS
  COMPLETED
}

// --- Islem Kaydi ---

model Transaction {
  id        String          @id @default(cuid())
  playerId  String          @map("player_id")
  type      TransactionType
  amount    Decimal         @db.Decimal(18, 9)
  txHash    String?         @map("tx_hash")
  status    TransactionStatus @default(PENDING)
  createdAt DateTime        @default(now()) @map("created_at")

  player Player @relation(fields: [playerId], references: [id])

  @@index([playerId])
  @@index([type])
  @@index([status])
  @@map("transactions")
}

enum TransactionType {
  ENTRY_FEE
  PAYOUT
  REFUND
  COMMISSION
}

enum TransactionStatus {
  PENDING
  CONFIRMED
  FAILED
}
```

**Schema Tasarim Kararlari:**

| Karar | Gerekcesi |
|-------|-----------|
| `cuid()` ID'ler | UUID'ye gore kisa, URL-guvenli, sirali (indeks dostu) |
| `Decimal(18,9)` para alanlari | Kripto para miktarlari icin yeterli hassasiyet (SOL/USDT) |
| `@map("snake_case")` | TypeScript'te camelCase, DB'de snake_case convention |
| `RoomPlayer` ara tablosu | Many-to-many iliski icin — bir oyuncu birden fazla odada olabilir |
| Enum'lar | State machine mantigi icin tip-guvenli status degerleri |
| Indeksler | Sik sorgulanan alanlarda (`status`, `roomId`, `playerId`) performans |

### 4. Redis Baglanti Modulu

#### 4.1 `backend/src/db/redis.ts`

```typescript
import Redis from 'ioredis';

const REDIS_URL = process.env.REDIS_URL || 'redis://localhost:6379';

export const redis = new Redis(REDIS_URL, {
  maxRetriesPerRequest: 3,
  retryStrategy(times) {
    const delay = Math.min(times * 200, 2000);
    return delay;
  },
});

redis.on('connect', () => {
  console.log('[Redis] Connected');
});

redis.on('error', (err) => {
  console.error('[Redis] Connection error:', err.message);
});
```

**Notlar:**
- `retryStrategy` — baglanti kesilirse artan aralikla yeniden dener (max 2sn)
- `maxRetriesPerRequest: 3` — tek bir sorgu icin max 3 deneme
- Singleton pattern — tum uygulama ayni Redis baglantisini kullanir

### 8. Backend Entry Point Guncellemesi

#### 8.1 `backend/src/index.ts`'e DB baglantisi ekleme

Mevcut sunucu baslatma koduna veritabani baglanti kontrolu eklenir:

```typescript
import { prisma } from './db/prisma.js';
import { redis } from './db/redis.js';

// ... mevcut Express + Socket.io kodu ...

// --- Start ---
const PORT = parseInt(process.env.PORT || '3001', 10);

async function start() {
  try {
    // Veritabani baglanti kontrolleri
    await prisma.$connect();
    console.log('[Prisma] PostgreSQL connected');

    await redis.ping();
    console.log('[Redis] Ping successful');

    httpServer.listen(PORT, () => {
      console.log(`[Server] Running on http://localhost:${PORT}`);
      console.log(`[Socket.io] WebSocket server ready`);
    });
  } catch (error) {
    console.error('[Server] Failed to start:', error);
    process.exit(1);
  }
}

// Graceful shutdown
process.on('SIGTERM', async () => {
  console.log('[Server] SIGTERM received, shutting down...');
  await prisma.$disconnect();
  redis.disconnect();
  httpServer.close();
  process.exit(0);
});

process.on('SIGINT', async () => {
  console.log('[Server] SIGINT received, shutting down...');
  await prisma.$disconnect();
  redis.disconnect();
  httpServer.close();
  process.exit(0);
});

start();
```

**Notlar:**
- `prisma.$connect()` — PostgreSQL baglantisinini dogrular
- `redis.ping()` — Redis'in erislebilir oldugunu dogrular
- Graceful shutdown — SIGTERM/SIGINT sinyallerinde baglantilari duzenli kapatir
- Baglanti hatasi durumunda sunucu baslamaz (`process.exit(1)`)

### 14. Redis Key Naming Convention (Dokumantasyon)

Faz 1'de Redis'e yazilacak veriler icin key isimlendirme standardi belirlenir. Bu task'ta kod yazilmaz, sadece convention tanimlanir:

```
room:{roomId}                → Hash: oda bilgileri (type, status, entryFee)
room:{roomId}:players        → Set: odadaki oyuncu ID'leri
room:{roomId}:state          → Hash: aktif oyun state'i (phase, round, timer)
player:{playerId}:session    → Hash: socket ID, baglanti zamani, aktif oda
game:{gameId}:round:{n}      → Hash: round state (selections, protected number)
game:{gameId}:positions      → Hash: oyuncu pozisyonlari (Format B)
```

**TTL Stratejisi:**
- Oda verileri: oyun bittikten 5 dakika sonra sil
- Session verileri: disconnect sonrasi 60 saniye grace period
- Pozisyon verileri: oyun bittikten hemen sonra sil

## Notlar ve Dikkat Edilecekler

1. **Docker Desktop gerekli**: macOS'ta Docker Compose calistirmak icin Docker Desktop kurulu olmali. Kurulu degilse `brew install --cask docker` ile kurulabilir.

2. **Port cakismasi**: PostgreSQL (5432) veya Redis (6379) portlari baska bir servis tarafindan kullaniliyorsa `docker-compose.yml`'deki port mapping'i degistir (ornegin `5433:5432`).

3. **Prisma Client cache**: `prisma generate` komutu `node_modules/.prisma/client` altina tip dosyalari olusturur. `node_modules` silindiginde `pnpm install` sonrasi `prisma generate` yeniden calistirilmalidir. Bu, `postinstall` script'ine eklenebilir.

4. **Decimal hassasiyeti**: `Decimal(18,9)` — toplam 18 basamak, noktadan sonra 9 basamak. Bu, Solana'nin SOL (9 decimals) ve cogu SPL token'in hassasiyetini kapsar. USDT (6 decimals) icin de yeterli.

5. **Migration vs db push**: Gelistirme sirasinda hizli prototipleme icin `db push` kullanilabilir, ancak production'a geciste tum degisiklikler migration ile yapilmali. Migration dosyalari versiyonlanir ve geri alinabilir.

6. **Redis veri kaliciligi**: Bu kurulumda Redis varsayilan ayarlarla calisir (RDB snapshot). Gelistirme ortaminda Redis verisi kaybolursa sorun olmaz — gecici veri zaten runtime'da yeniden olusturulur.

7. **Connection pooling**: Prisma varsayilan olarak connection pooling yapar (num_connections = num_cpus * 2 + 1). Production'da PgBouncer eklenecek (Faz 4, Task 12.x).
