| Ozellik | Aciklama |

## Mimari Kararlar

### Paylasilmis Tipler Yok
Frontend ve backend kendi tiplerini BAGIMSIZ olarak tanimlar. `shared/` paketi YOKTUR ve OLUSTURULMAMALIDIR.
- Frontend tipleri: `frontend/src/types/`
- Backend tipleri: `backend/src/types/`
- Socket event isimleri her iki tarafta sabit olarak tanimlidir

### Veritabani Stratejisi
- **PostgreSQL:** Kalici veri (oyun gecmisi, islemler, oyuncu kayitlari)
- **Redis:** Gecici veri (aktif oyun state'i, oda bilgileri, oyuncu pozisyonlari)
- Ayni veriyi iki veritabaninda TUTMA

### Monorepo
- pnpm workspaces + Turborepo
- Paketler: frontend, backend, contracts
- Root'tan calistirma: `pnpm dev`, `pnpm build`, `pnpm lint`
- Tek paket filtreleme: `pnpm --filter @web3-gaming/frontend dev`

## Kod Kurallari

### Genel
- TypeScript strict mode — `any` kullanma, tip tanimla
- Dosya isimlendirme: kebab-case (`use-game-store.ts`, `validate-env.ts`)
- Import alias: `@/` → `src/` (her iki pakette)
- Prettier formatlamasi zorunlu (pre-commit hook ile)
- Satir uzunlugu: 100 karakter
- Tek tirnak, noktali virgul, trailing comma

### Frontend
- React fonksiyonel bilesenler — class component KULLANMA
- State yonetimi: Zustand (Redux, MobX, Context KULLANMA)
- Styling: Tailwind CSS utility class'lari (styled-components, CSS modules KULLANMA)
- 3D rendering: React Three Fiber + Three.js (Format B icin)
- Animasyon: GSAP (Format A ok animasyonu icin)
- Routing: React Router v7

### Backend
- Express 5 + Socket.io
- Veritabani erisimi: Prisma ORM (raw SQL KULLANMA, zorunlu olmadikca)
- Redis erisimi: ioredis
- Ortam degiskenleri: `validateEnv()` fonksiyonu ile dogrula, `process.env` dogrudan kullanma
- Logging prefix formati: `[Modul] Mesaj` (ornek: `[Socket] Connected: abc123`)
- Graceful shutdown: SIGTERM ve SIGINT handle et

### Smart Contracts
- Rust + Anchor Framework
- Program: `contracts/programs/escrow/`
- Test: TypeScript + Mocha (`contracts/tests/`)
- Build: `anchor build` (Turborepo uzerinden)

## Yasaklar

- `shared/` paketi veya dizini olusturma
- `any` tipi kullanma (zorunluysa `unknown` + type guard tercih et)
- `console.log` yerine yapilandirilmis logging kullan (production'da)

## Onemli Notlar

### CLAUDE.md Anti-Pattern'leri

Kacinilmasi gereken yayginn hatalar:

| Anti-Pattern | Neden Kotu | Dogru Yaklasim |
|-------------|-----------|----------------|
| 500+ satirlik CLAUDE.md | Context window'u doldurur, diger iceriklere yer kalmaz | 200 satir limiti, detaylar baska dosyalarda |
| Kod snippet'leri iceren CLAUDE.md | Kod degisince CLAUDE.md guncel kalmaz | "X dosyasina bak" diye referans ver |
| Her PR'da CLAUDE.md guncelleme | Gereksiz diff, review yuku | Sprint sonunda toplu gozden gecirme |
| Belirsiz kurallar | Claude "mumkunse" ifadesini bazen yapar bazen yapmaz | Kesin kural: "kullan" veya "KULLANMA" |
| Root'ta alt dizin kurallarini tekrarlama | Context window israfi, tutarsizlik riski | Hiyerarsi: root = genel, alt dizin = ozel |
| Dependency versiyonlari listeleme | `package.json` degisince eski kalir | "Versiyonlar icin `package.json`'a bak" |
