## Amac

`backend/` workspace paketine production-grade bir Node.js sunucu ortami kurmak. Express ile HTTP API, Socket.io ile real-time WebSocket iletisimi, TypeScript ile tip guvenligi ve Nodemon ile hot-reload destekli gelistirme ortami saglanir.

Bu task tamamlandiginda backend paketi `pnpm dev` ile calisir durumda olacak, `localhost:3001`'de HTTP ve WebSocket baglantilari kabul edecek ve sonraki fazlardaki oda yonetimi, oyun mantigi ve blockchain entegrasyonu icin saglam bir temel olusturacak.

### 2. Express + Socket.io Sunucu Kurulumu

#### 2.1 `backend/src/index.ts`

```typescript
import 'dotenv/config';
import express from 'express';
import { createServer } from 'http';
import { Server } from 'socket.io';
import cors from 'cors';
import helmet from 'helmet';

const app = express();
const httpServer = createServer(app);

const io = new Server(httpServer, {
  cors: {
    origin: process.env.FRONTEND_URL || 'http://localhost:5173',
    methods: ['GET', 'POST'],
  },
});

// --- Middleware ---
app.use(helmet());
app.use(cors({
  origin: process.env.FRONTEND_URL || 'http://localhost:5173',
}));
app.use(express.json());

// --- Health Check ---
app.get('/api/health', (_req, res) => {
  res.json({ status: 'ok', timestamp: Date.now() });
});

// --- Socket.io ---
io.on('connection', (socket) => {
  console.log(`[Socket] Connected: ${socket.id}`);

  socket.on('disconnect', (reason) => {
    console.log(`[Socket] Disconnected: ${socket.id} (${reason})`);
  });
});

// --- Start ---
const PORT = parseInt(process.env.PORT || '3001', 10);

httpServer.listen(PORT, () => {
  console.log(`[Server] Running on http://localhost:${PORT}`);
  console.log(`[Socket.io] WebSocket server ready`);
});

export { app, httpServer, io };
```

**Notlar:**
- `createServer(app)` ile HTTP sunucusu olusturulur — Express ve Socket.io ayni port'u paylasir
- CORS ayari frontend dev server (`localhost:5173`) ile uyumlu
- Health check endpoint'i (`/api/health`) sunucunun ayakta oldugunu dogrulamak icin
- Socket.io baglanti/ayrilma loglari gelistirme sirasinda debug icin faydali
- `export` ile test ve ilerideki modullerde kullanim icin erisim saglanir

```json
{
  "extends": "../tsconfig.json",
  "compilerOptions": {
    "composite": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    },
    "module": "ESNext",
    "moduleResolution": "bundler",
    "esModuleInterop": true,
    "resolveJsonModule": true
  },
  "include": ["src/**/*.ts"],
```

**Notlar:**
- `module: "ESNext"` ve `moduleResolution: "bundler"` — ESM import/export destegi (`"type": "module"` ile uyumlu)
- `esModuleInterop` — CommonJS modulleri (express, cors vb.) default import ile kullanilabilir
- `resolveJsonModule` — JSON dosyalarini import etme destegi (config dosyalari icin)

```json
{

    "dev": "tsx watch src/index.ts",
```

| Script | Aciklama |

| `lint` | ESLint ile kod kontrolu |
| `type-check` | TypeScript tip kontrolu (build olmadan) |
| `format` | Prettier ile tum kaynak dosyalari formatla |
| `format:check` | CI icin format kontrolu |

**Notlar:**
- `tsx watch` nodemon + ts-node kombinasyonunun modern ve hizli alternatifi
- `start` script'i production ortaminda kullanilir (build sonrasi)

## Notlar ve Dikkat Edilecekler

1. **ESM Uyumlulugu**: Proje `"type": "module"` kullandiginden tum importlar ESM formatinda olmali. CommonJS modulleri (express vb.) `esModuleInterop` sayesinde default import ile kullanilabilir.

2. **tsx vs ts-node vs nodemon**: `tsx` secilmesinin nedeni — ts-node'a gore cok daha hizli baslatma suresi (esbuild tabanli), nodemon'a ek bagimlilik gerektirmez, ve ESM destegi sorunsuz.

3. **Express v5**: Express 5 async error handling icin native destek saglar. `async` route handler'larda hata frlatildiginda otomatik olarak error middleware'e iletilir.

4. **Socket.io CORS**: Socket.io'nun kendi CORS ayari var — Express CORS middleware'i Socket.io'yu kapsamaz. Her ikisini de ayri yapilandirmak gerekir.

5. **Port Secimi**: `3001` portu Milestone M0 ile uyumlu. Frontend Vite proxy ayarlari (`/api` → `localhost:3001`, `/socket.io` → `localhost:3001`) Task 0.2'de zaten yapilandirildi.

6. **Prisma henuz yok**: Veritabani baglantisi bu task'ta kurulmaz. Task 0.4 (Veritabani Kurulumu) PostgreSQL + Redis + Prisma ORM'i yapilandiracak.
