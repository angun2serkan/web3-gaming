# Task 0.8 — Ortam Degiskenleri, Docker Template'leri ve Gitignore Kontrolu

**Faz:** 0 — Proje Altyapisi
**Tahmini Efor:** 0.5 gun
**Durum:** Beklemede
**Bagimlilik:** Task 0.1–0.7 (Monorepo, Frontend, Backend, Database, Contracts, CI/CD, Types) — tamamlanmis olmali

---

## Amac

Projede development, staging ve production ortam degiskenlerini standartlastirmak. Her ortam icin `.env.example` template'leri, Docker Compose profilleri, uretim ortami icin Dockerfile'lar ve `.dockerignore` dosyalari hazirlamak. Mevcut `.gitignore` dosyasini projenin guncel ihtiyaclariyla kontrol edip guncellemek.

Bu task tamamlandiginda:
- Backend ve frontend icin tum ortam degiskenleri tanimlanmis ve dokumante edilmis olacak
- `.env.example` dosyalari yeni bir gelistiricinin `cp .env.example .env` ile hemen calismaya baslamasini saglayacak
- `docker-compose.yml` local gelistirme icin, `docker-compose.prod.yml` VPS/production icin hazir olacak
- Backend ve frontend icin multi-stage Dockerfile'lar yazilmis olacak
- `.gitignore` projenin tum paketleriyle (frontend, backend, contracts, Docker) uyumlu olacak
- VPS'e gecis icin sadece `.env` dosyalarinin olusturulmasi ve `docker compose up` komutunun yeterli olmasi hedeflenir

> **Not:** Bu task'ta gercek deploy yapilmaz. Sadece yapilandirma dosyalari hazirlanir. Gercek VPS provisioning ve deploy Faz 3-4'te yapilacaktir.

### .gitignore

Mevcut `.gitignore` temel kurallari kapsiyor ancak asagidaki eksiklikler tespit edildi:

| Eksik Kural | Risk |
|-------------|------|
| `.env.production` / `.env.staging` | Production secret'lari yanlislikla commit edilebilir |
| `backend/prisma/*.db` | SQLite fallback dosyalari track edilebilir |
| `*.tsbuildinfo` | TypeScript incremental build cache dosyalari |
| `coverage/` | Test coverage raporlari |
| `*.pem` / `*.key` | SSL sertifika/key dosyalari |
| `.vercel/` / `.netlify/` | Deploy platform cache'leri |

```bash
# Degerler: localnet | devnet | mainnet-beta

# Platform cuzdan adresi (komisyon alacak cuzdan)

PLATFORM_WALLET_PRIVATE_KEY=

# JWT veya session secret (ileride auth icin)
JWT_SECRET=
# Rate limiting (istekler/dakika)
RATE_LIMIT_MAX=100
RATE_LIMIT_WINDOW_MS=60000

# --- Logging ---
# Degerler: debug | info | warn | error

```

### 2. Frontend Ortam Degiskenleri

Vite'ta ortam degiskenleri `VITE_` prefix'i ile tanimlanir. Sadece `VITE_` ile baslayan degiskenler client bundle'a dahil edilir.

#### 2.1 `frontend/.env.example`

```bash
# ===================================
# WEB3 GAMING PLATFORM — Frontend
# ===================================
# Bu dosyayi kopyalayip .env olarak yeniden adlandirin:
#   cp .env.example .env

# --- API Baglantisi ---
# Development: bos birakilir (Vite proxy kullanilir)
# Staging/Production: Backend URL
VITE_API_URL=
# Ornek: https://api.staging.example.com

# --- WebSocket Baglantisi ---
# Development: bos birakilir (Vite proxy kullanilir)
# Staging/Production: Backend WebSocket URL
VITE_WS_URL=
# Ornek: https://api.staging.example.com

# --- Solana ---
VITE_SOLANA_NETWORK=devnet
# Degerler: localnet | devnet | mainnet-beta

VITE_SOLANA_RPC_URL=
# Development: bos (varsayilan public endpoint)
# Production: Ozel RPC endpoint onerilir (Helius, QuickNode vb.)

# --- Uygulama ---
VITE_APP_NAME=Web3 Gaming Platform
VITE_COMMISSION_RATE=0.05
```

#### 2.2 `frontend/.env` (Development — gitignore'da)

```bash
VITE_API_URL=
VITE_WS_URL=
VITE_SOLANA_NETWORK=devnet
VITE_SOLANA_RPC_URL=
VITE_APP_NAME=Web3 Gaming Platform (Dev)
VITE_COMMISSION_RATE=0.05
```

**Notlar:**
- Development'ta `VITE_API_URL` ve `VITE_WS_URL` bos birakilir — `vite.config.ts` icindeki proxy yapilandirmasi devreye girer
- Production'da proxy yok — bu degiskenler gercek backend URL'lerini gostermelidir
- `VITE_` prefix'i olmayan degiskenler client bundle'a **dahil edilmez** — bu guvenlik icin onemlidir
- **Hassas bilgiler (secret key, private key) asla `VITE_` ile tanimlanmamali** — client tarafinda gorunur olur

### 4. Docker Compose — Development (Guncelleme)

Mevcut `docker-compose.yml` root `.env` dosyasindan degiskenleri okuyacak sekilde guncellenir.

#### 4.1 `docker-compose.yml` (Guncellenmis)

```yaml
# ===================================
# Development: sadece veritabanlari
# Kullanim: docker compose up -d
# ===================================

services:
  postgres:
    image: postgres:16-alpine
    container_name: web3gaming-postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: ${POSTGRES_USER:-web3gaming}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-web3gaming_dev}
      POSTGRES_DB: ${POSTGRES_DB:-web3gaming}
    ports:
      - "${POSTGRES_PORT:-5432}:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-web3gaming}"]
      interval: 5s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: web3gaming-redis
    restart: unless-stopped
    ports:
      - "${REDIS_PORT:-6379}:6379"
    volumes:
      - redis_data:/data
    command: redis-server --appendonly yes
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  postgres_data:
  redis_data:
```

**Degisiklikler:**
- Environment degiskenleri `${VAR:-default}` syntax'i ile root `.env`'den okunur
- Fallback degerler mevcut — `.env` dosyasi olmadan da calisir
- Redis'e `--appendonly yes` eklendi — veri kaliciligi (restart sonrasi kaybolmaz)

### 5. Docker Compose — Production

#### 5.1 `docker-compose.prod.yml`

```yaml
# ===================================
# Production / Staging: tam uygulama stack'i
# Kullanim: docker compose -f docker-compose.prod.yml up -d
# ===================================

services:
  # --- Veritabanlari ---

  postgres:
    image: postgres:16-alpine
    container_name: web3gaming-postgres
    restart: always
    environment:
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: ${POSTGRES_DB}
    ports:
      - "127.0.0.1:${POSTGRES_PORT:-5432}:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER}"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: web3gaming-redis
    restart: always
    ports:
      - "127.0.0.1:${REDIS_PORT:-6379}:6379"
    volumes:
      - redis_data:/data
    command: redis-server --appendonly yes --requirepass ${REDIS_PASSWORD:-}
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD:-}", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  # --- Backend ---

  backend:
    build:
      context: .
      dockerfile: backend/Dockerfile
    container_name: web3gaming-backend
    restart: always
    ports:
      - "127.0.0.1:${BACKEND_PORT:-3001}:3001"
    environment:
      PORT: 3001
      NODE_ENV: ${NODE_ENV:-production}
      FRONTEND_URL: ${FRONTEND_URL}
      DATABASE_URL: postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@postgres:5432/${POSTGRES_DB}
      REDIS_URL: redis://:${REDIS_PASSWORD:-}@redis:6379
      SOLANA_RPC_URL: ${SOLANA_RPC_URL}
      SOLANA_NETWORK: ${SOLANA_NETWORK}
      PLATFORM_WALLET_ADDRESS: ${PLATFORM_WALLET_ADDRESS}
      PLATFORM_WALLET_PRIVATE_KEY: ${PLATFORM_WALLET_PRIVATE_KEY}
      JWT_SECRET: ${JWT_SECRET}
      RATE_LIMIT_MAX: ${RATE_LIMIT_MAX:-100}
      RATE_LIMIT_WINDOW_MS: ${RATE_LIMIT_WINDOW_MS:-60000}
      LOG_LEVEL: ${LOG_LEVEL:-info}
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy

  # --- Frontend (Nginx ile static serve) ---

  frontend:
    build:
      context: .
      dockerfile: frontend/Dockerfile
      args:
        VITE_API_URL: ${VITE_API_URL}
        VITE_WS_URL: ${VITE_WS_URL}
        VITE_SOLANA_NETWORK: ${VITE_SOLANA_NETWORK}
        VITE_SOLANA_RPC_URL: ${VITE_SOLANA_RPC_URL}
        VITE_APP_NAME: ${VITE_APP_NAME}
        VITE_COMMISSION_RATE: ${VITE_COMMISSION_RATE}
    container_name: web3gaming-frontend
    restart: always
    ports:
      - "${FRONTEND_PORT:-80}:80"
    depends_on:
      - backend

volumes:
  postgres_data:
  redis_data:
```

**Onemli Farkliliklar (dev vs prod):**

| Ozellik | Development (`docker-compose.yml`) | Production (`docker-compose.prod.yml`) |
|---------|-----------------------------------|---------------------------------------|
| Servisler | Sadece PostgreSQL + Redis | PostgreSQL + Redis + Backend + Frontend |
| Port binding | `0.0.0.0` (tum arayuzler) | `127.0.0.1` (sadece localhost — reverse proxy arkasinda) |
| Restart policy | `unless-stopped` | `always` |
| Redis sifresi | Yok | `REDIS_PASSWORD` ile korunur |
| DB baglantisi | `localhost` uzerinden | Docker network uzerinden (`postgres:5432`) |
| Frontend | Vite dev server (HMR) | Nginx ile static build serve |
| Backend | `tsx watch` (hot reload) | Derlanmis `node dist/index.js` |
| Env degiskenleri | Fallback degerler mevcut | Fallback yok — `.env` dosyasi zorunlu |

### 6. Backend Dockerfile

#### 6.1 `backend/Dockerfile`

```dockerfile
# ===================================
# Backend — Multi-stage Build
# ===================================

# --- Stage 1: Dependency Installation ---
FROM node:22-alpine AS deps
WORKDIR /app

# pnpm kur
RUN corepack enable && corepack prepare pnpm@9.15.4 --activate

# Sadece dependency dosyalarini kopyala (cache katmani)
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY backend/package.json backend/
COPY frontend/package.json frontend/
COPY contracts/package.json contracts/

# Sadece backend bagimliklarini kur
RUN pnpm install --frozen-lockfile --filter @web3-gaming/backend...

# --- Stage 2: Build ---
FROM node:22-alpine AS builder
WORKDIR /app

RUN corepack enable && corepack prepare pnpm@9.15.4 --activate

# Dependency'leri onceki stage'den kopyala
COPY --from=deps /app/node_modules ./node_modules
COPY --from=deps /app/backend/node_modules ./backend/node_modules

# Kaynak kodu kopyala
COPY backend/ backend/
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml tsconfig.json ./

# Prisma client uret
RUN cd backend && npx prisma generate

# TypeScript derle
RUN cd backend && npx tsc

# --- Stage 3: Production Runtime ---
FROM node:22-alpine AS runner
WORKDIR /app

RUN corepack enable && corepack prepare pnpm@9.15.4 --activate

# Guvenlik: non-root kullanici
RUN addgroup --system --gid 1001 nodejs && \
    adduser --system --uid 1001 backend
USER backend

# Sadece production icin gerekli dosyalari kopyala
COPY --from=builder --chown=backend:nodejs /app/backend/dist ./dist
COPY --from=builder --chown=backend:nodejs /app/backend/package.json ./
COPY --from=builder --chown=backend:nodejs /app/backend/node_modules ./node_modules
COPY --from=builder --chown=backend:nodejs /app/backend/prisma ./prisma

# Port
EXPOSE 3001

# Healthcheck
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:3001/api/health || exit 1

# Baslat
CMD ["node", "dist/index.js"]
```

**Notlar:**
- **Multi-stage build** — 3 asamali: dependency kurulumu → TypeScript derleme → production runtime
- **Non-root kullanici** — guvenlik icin `backend` kullanicisi olusturulur
- **Layer cache** — `package.json` ve `pnpm-lock.yaml` onceden kopyalanir, kaynak kodu degisince dependency'ler yeniden kurulmaz
- **Prisma client** — build asamasinda `prisma generate` ile uretilir
- **Healthcheck** — `/api/health` endpoint'i uzerinden konteyner sagligi izlenir
- Alpine imaj — kucuk boyut (~150MB vs ~1GB full Node imaji)

### 7. Frontend Dockerfile

#### 7.1 `frontend/Dockerfile`

```dockerfile
# ===================================
# Frontend — Multi-stage Build + Nginx
# ===================================

# --- Stage 1: Dependency Installation ---
FROM node:22-alpine AS deps
WORKDIR /app

RUN corepack enable && corepack prepare pnpm@9.15.4 --activate

COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY frontend/package.json frontend/
COPY backend/package.json backend/
COPY contracts/package.json contracts/

RUN pnpm install --frozen-lockfile --filter @web3-gaming/frontend...

# --- Stage 2: Build ---
FROM node:22-alpine AS builder
WORKDIR /app

RUN corepack enable && corepack prepare pnpm@9.15.4 --activate

COPY --from=deps /app/node_modules ./node_modules
COPY --from=deps /app/frontend/node_modules ./frontend/node_modules

COPY frontend/ frontend/
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml tsconfig.json ./

# Vite build — VITE_ env degiskenleri build-time'da bundle'a yazilir
ARG VITE_API_URL
ARG VITE_WS_URL
ARG VITE_SOLANA_NETWORK
ARG VITE_SOLANA_RPC_URL
ARG VITE_APP_NAME
ARG VITE_COMMISSION_RATE

RUN cd frontend && npx vite build

# --- Stage 3: Nginx ile Serve ---
FROM nginx:1.27-alpine AS runner

# Nginx konfigurasyonu
COPY frontend/nginx.conf /etc/nginx/conf.d/default.conf

# Build ciktisini nginx'e kopyala
COPY --from=builder /app/frontend/dist /usr/share/nginx/html

# Port
EXPOSE 80

# Healthcheck
HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:80/ || exit 1

CMD ["nginx", "-g", "daemon off;"]
```

**Not:** Vite'in ortam degiskenleri **build-time'da** bundle'a yazilir. Bu nedenle `ARG` olarak alinir ve `docker compose build` sirasinda `.env` dosyasindan okunur.

#### 7.2 `frontend/nginx.conf`

```nginx
server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    # Gzip sikistirma
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript image/svg+xml;

    # Statik dosyalar icin cache
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # API isteklerini backend'e yonlendir
    location /api/ {
        proxy_pass http://backend:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket isteklerini backend'e yonlendir
    location /socket.io/ {
        proxy_pass http://backend:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # SPA routing — tum bilinmeyen path'ler index.html'e yonlendirilir
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

**Notlar:**
- SPA routing: React Router'in client-side routing'i icin tum bilinmeyen path'ler `index.html`'e yonlendirilir
- Gzip: JavaScript, CSS ve diger metin dosyalari sikistirilir — bant genisligi tasarrufu
- Static cache: Vite'in hash'li dosyalari 1 yil cache'lenir — `immutable` header'i ile
- API/Socket proxy: Nginx icinden backend konteynerine yonlendirme (Docker network uzerinden `backend:3001`)

### 10. Ortam Degiskeni Dogrulama (Startup Validation)

Backend baslatildiginda tum zorunlu ortam degiskenlerinin tanimli oldugu kontrol edilmelidir. Eksik degisken varsa sunucu acik hata mesajiyla baslamayi reddeder.

#### 10.1 `backend/src/utils/validate-env.ts`

```typescript
interface EnvConfig {
  PORT: number;
  NODE_ENV: 'development' | 'staging' | 'production';
  FRONTEND_URL: string;
  DATABASE_URL: string;
  REDIS_URL: string;
  // Opsiyonel — sadece staging/production'da zorunlu
  SOLANA_RPC_URL?: string;
  SOLANA_NETWORK?: string;
  PLATFORM_WALLET_ADDRESS?: string;
  PLATFORM_WALLET_PRIVATE_KEY?: string;
  JWT_SECRET?: string;
  RATE_LIMIT_MAX?: number;
  RATE_LIMIT_WINDOW_MS?: number;
  LOG_LEVEL?: string;
}

/** Her ortam icin zorunlu degiskenler */
const REQUIRED_VARS: Record<string, string[]> = {
  // Her ortamda zorunlu
  all: ['PORT', 'NODE_ENV', 'FRONTEND_URL', 'DATABASE_URL', 'REDIS_URL'],
  // Sadece production'da zorunlu
  production: ['JWT_SECRET', 'SOLANA_RPC_URL', 'PLATFORM_WALLET_ADDRESS'],
  // Sadece staging'de zorunlu
  staging: ['JWT_SECRET', 'SOLANA_RPC_URL'],
};

export function validateEnv(): EnvConfig {
  const env = process.env;
  const nodeEnv = env.NODE_ENV || 'development';
  const missing: string[] = [];

  // Tum ortamlar icin zorunlu degiskenler
  for (const key of REQUIRED_VARS.all) {
    if (!env[key]) missing.push(key);
  }

  // Ortam bazli zorunlu degiskenler
  const envSpecific = REQUIRED_VARS[nodeEnv] || [];
  for (const key of envSpecific) {
    if (!env[key]) missing.push(key);
  }

  if (missing.length > 0) {
    console.error(`\n[ENV] Eksik ortam degiskenleri (${nodeEnv}):`);
    missing.forEach((key) => console.error(`  - ${key}`));
    console.error(`\nCozum: backend/.env dosyasini kontrol edin.\n`);
    process.exit(1);
  }

  return {
    PORT: parseInt(env.PORT || '3001', 10),
    NODE_ENV: nodeEnv as EnvConfig['NODE_ENV'],
    FRONTEND_URL: env.FRONTEND_URL!,
    DATABASE_URL: env.DATABASE_URL!,
    REDIS_URL: env.REDIS_URL!,
    SOLANA_RPC_URL: env.SOLANA_RPC_URL,
    SOLANA_NETWORK: env.SOLANA_NETWORK,
    PLATFORM_WALLET_ADDRESS: env.PLATFORM_WALLET_ADDRESS,
    PLATFORM_WALLET_PRIVATE_KEY: env.PLATFORM_WALLET_PRIVATE_KEY,
    JWT_SECRET: env.JWT_SECRET,
    RATE_LIMIT_MAX: env.RATE_LIMIT_MAX ? parseInt(env.RATE_LIMIT_MAX, 10) : undefined,
    RATE_LIMIT_WINDOW_MS: env.RATE_LIMIT_WINDOW_MS ? parseInt(env.RATE_LIMIT_WINDOW_MS, 10) : undefined,
    LOG_LEVEL: env.LOG_LEVEL,
  };
}
```

#### 10.2 `backend/src/index.ts` Guncelleme

Dosyanin basina eklenir:

```typescript
import 'dotenv/config';
import { validateEnv } from './utils/validate-env.js';

const env = validateEnv();

// Ardindan env.PORT, env.FRONTEND_URL vb. kullanilir
// process.env yerine tip-guvenli env objesi tercih edilir
```

**Notlar:**
- `validateEnv()` basarisiz olursa `process.exit(1)` ile sunucu baslamaz — eksik degiskenler acikca listelenir
- Development'ta sadece temel degiskenler zorunlu (DB, Redis)
- Production'da Solana ve JWT degiskenleri de zorunlu
- Fonksiyon tip-guvenli `EnvConfig` objesi dondurur — `process.env.PORT` yerine `env.PORT` kullanilir

### 11. Docker Compose Kullanim Rehberi

#### Development (Yerel Gelistirme)

```bash
# Veritabanlarini baslat
docker compose up -d

# Uygulamayi baslat (Turborepo ile)
pnpm dev

# Veritabanlarini durdur
docker compose down

# Veritabanlarini sifirla (tum veri silinir)
docker compose down -v && docker compose up -d
```

#### Staging / Production (VPS)

```bash
# 1. .env dosyasini olustur (root dizinde)
cp .env.example .env
# Degerleri VPS ortamina gore duzenle (gercek sifre, domain, RPC URL vb.)

# 2. Tum servisleri build et ve baslat
docker compose -f docker-compose.prod.yml up -d --build

# 3. Veritabani migration'larini calistir
docker compose -f docker-compose.prod.yml exec backend npx prisma migrate deploy

# 4. Loglari izle
docker compose -f docker-compose.prod.yml logs -f

# 5. Tek bir servisi yeniden baslat
docker compose -f docker-compose.prod.yml restart backend

# 6. Tum servisleri durdur
docker compose -f docker-compose.prod.yml down
```

## Onemli Notlar

### Guvenlik Kurallari

| Kural | Aciklama |
|-------|----------|
| `.env` dosyalari git'e **asla** commit edilmez | Sadece `.env.example` commit edilir |
| `VITE_` prefix'li degiskenlere secret **koymlayin** | Client bundle'da gorunur — herkes okuyabilir |
| Production sifreleri en az **32 karakter** rastgele string olmali | `openssl rand -base64 32` ile uretilebilir |
| `PLATFORM_WALLET_PRIVATE_KEY` en hassas deger | VPS'te bile env var yerine secret manager (Vault, AWS Secrets) onerilir |
| Redis production'da **sifre ile korunmali** | `--requirepass` flag'i docker-compose.prod.yml'de tanimli |
| PostgreSQL production'da **disariya acik olmamali** | `127.0.0.1:5432` binding ile sadece localhost'tan erisim |

### VPS Hazirlik Notlari

VPS'e gecis icin bu task'in urettigii dosyalarin yaninda asagidakiler de gerekecektir (bu task'in kapsaminda **degil**, ancak bilgi amaciyla):

| Ihtiyac | Aciklama | Ne Zaman |
|---------|----------|----------|
| **Domain ve DNS** | A record → VPS IP | VPS provisioningg oncesi |
| **SSL sertifikasi** | Let's Encrypt (certbot) veya Cloudflare | Deploy oncesi |
| **Reverse proxy** | Nginx veya Caddy — VPS host uzerinde, Docker disinda | Deploy oncesi |
| **Firewall** | UFW ile sadece 80, 443, 22 portlarini ac | VPS provisioning |
| **Docker kurulumu** | Docker Engine + Docker Compose plugin | VPS provisioning |
| **Monitoring** | Uptime monitoring (UptimeRobot, Grafana) | Deploy sonrasi |
| **Backup** | PostgreSQL otomatik yedekleme (cron + pg_dump) | Deploy sonrasi |
| **CI/CD deploy** | GitHub Actions ile otomatik VPS deploy | Faz 3-4 |

### Bu Task'ta Yapilmayanlar

| Konu | Aciklama | Ne Zaman |
|------|----------|----------|
| Gercek VPS provisioning | Sunucu kurulumu, Docker yukleme, firewall | Faz 3+ |
| SSL/TLS yapilandirmasi | Let's Encrypt sertifikasi, HTTPS | Faz 3+ |
| CI/CD deploy pipeline | GitHub Actions → VPS otomatik deploy | Faz 4 |
| Secret management | HashiCorp Vault veya AWS Secrets Manager | Faz 4 (production oncesi) |
| Horizontal scaling | Docker Swarm veya Kubernetes | Post-launch |
| Log aggregation | ELK Stack veya Grafana Loki | Faz 4 |
