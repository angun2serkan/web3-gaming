# Task 0.2 — Frontend Boilerplate

**Faz:** 0 — Proje Altyapisi
**Tahmini Efor:** 1 gun
**Durum:** Beklemede
**Bagimlilik:** Task 0.1 (Monorepo Yapisi) — tamamlanmis olmali

---

## Amac

`frontend/` workspace paketine production-grade bir React gelistirme ortami kurmak. Vite ile hizli dev server ve build, React 18+ ile UI, TypeScript ile tip guvenligi, Tailwind CSS ile utility-first styling, React Three Fiber ile 3D rendering altyapisi (Format B — Snake Arena icin), ve ESLint + Prettier ile kod kalitesi standartlari saglenir.

Bu task tamamlandiginda frontend paketi `pnpm dev` ile calisir durumda olacak ve sonraki fazlardaki UI gelistirmesi icin saglam bir temel olusturacak.

---

- `frontend/package.json` — minimal, sadece TypeScript bagimliligi var

### 2. Vite Yapilandirmasi

#### 2.1 `frontend/vite.config.ts`

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import tailwindcss from '@tailwindcss/vite';
import path from 'path';

export default defineConfig({
  plugins: [
    react(),
    tailwindcss(),
  ],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:3001',
        changeOrigin: true,
      },
      '/socket.io': {
        target: 'http://localhost:3001',
        ws: true,
      },
    },
  },
  build: {
    outDir: 'dist',
    sourcemap: true,
  },
});
```

**Notlar:**
- `@` path alias: `import { Button } from '@/components/Button'` seklinde kullanim saglar
- Proxy ayarlari: Backend API ve WebSocket baglantilari dev server uzerinden yonlendirilir (CORS sorunu onlenir)
- `server.port: 5173`: Milestone M0'daki hedefle uyumlu

### 4. ESLint Yapilandirmasi

#### 4.1 `frontend/eslint.config.js`

```javascript
import js from '@eslint/js';
import tseslint from 'typescript-eslint';
import reactHooks from 'eslint-plugin-react-hooks';
import reactRefresh from 'eslint-plugin-react-refresh';
import prettierConfig from 'eslint-config-prettier';

export default tseslint.config(
  { ignores: ['dist'] },
  {
    extends: [js.configs.recommended, ...tseslint.configs.recommended],
    files: ['**/*.{ts,tsx}'],
    plugins: {
      'react-hooks': reactHooks,
      'react-refresh': reactRefresh,
    },
    rules: {
      ...reactHooks.configs.recommended.rules,
      'react-refresh/only-export-components': [
        'warn',
        { allowConstantExport: true },
      ],
      '@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }],
    },
  },
  prettierConfig,
);
```

**Kurallar:**
- React Hooks kurallari zorunlu (`exhaustive-deps`, `rules-of-hooks`)
- React Refresh uyumlulugu (HMR'in duzgun calismasi icin)
- Kullanilmayan degiskenler hata verir (`_` prefix'li parametreler haric)
- Prettier ile cakisma olmamasi icin `eslint-config-prettier` en sonda

#### 7.2 `frontend/src/App.tsx`

```tsx
import { Routes, Route } from 'react-router';

export function App() {
```

## Notlar ve Dikkat Edilecekler

1. **Tailwind v4**: Eski `tailwind.config.js` yaklasimi yerine CSS-native `@theme` direktifi kullanilir. Eger proje ileride v3'e donmek zorunda kalirsa bu kisim degistirilmeli.

2. **React Three Fiber performansi**: R3F komponentleri performans acisindan hassastir. Canvas disindaki state degisiklikleri Canvas'i re-render etmemelidir. Zustand ile R3F state ayirmasi Faz 1'de detaylandirilacak.

3. **Mobile-first**: Tum Tailwind siniflari mobile-first yazilmali (`md:`, `lg:` prefix'leri buyuk ekranlar icin). Bu proje overview'daki "mobile-first tasarim" karariyla uyumludur.

4. **Proxy ayarlari**: Dev ortaminda frontend `localhost:5173`, backend `localhost:3001`'de calisir. Vite proxy sayesinde frontend'den `/api/*` ve `/socket.io/*` istekleri backend'e yonlendirilir.
